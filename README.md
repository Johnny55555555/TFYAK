# 3. Постановка задачи

Необходимо разработать **лексический анализатор**, который выполняет следующие функции:

* принимает текст программы из области редактирования;
* последовательно анализирует входную строку;
* выделяет допустимые лексемы;
* классифицирует лексемы по типам;
* определяет местоположение лексемы в тексте;
* обнаруживает недопустимые символы;
* выводит результат анализа в таблицу.

Лексический анализатор реализуется в виде отдельного программного модуля **LexicalAnalyzer**, который возвращает список найденных лексем.

Результаты анализа отображаются в таблице **DataGrid**, содержащей:

* условный код лексемы;
* тип лексемы;
* текст лексемы;
* местоположение в исходном тексте.

Также реализована навигация по ошибкам: при выборе строки таблицы курсор редактора устанавливается на позицию соответствующей лексемы.

# 8. Описание работы автомата

Работа сканера основана на принципе **конечного автомата**.

Алгоритм работы:

1. Автомат находится в **начальном состоянии**.
2. Считывается очередной символ входной строки.
3. В зависимости от типа символа выполняется переход в новое состояние:

   * буква → начало идентификатора;
   * цифра → начало числа;
   * символ `"` → начало строковой константы;
   * символы `[ ] = ;` → отдельные лексемы;
   * пробел → разделитель.
4. Когда лексема полностью распознана, она добавляется в таблицу результатов.
5. Если встречается недопустимый символ, генерируется лексема типа **ERROR**.
6. После обработки символа автомат возвращается в начальное состояние.
